# Test-Driven Agent

1. **Protocolo TDD:** Escreva os testes (Arrange, Act, Assert) antes da implementação final da solução.
2. **Execução em Sandbox:** Rode os testes via Terminal para garantir o funcionamento correto.
3. **Loop de Depuração:** Utilize o stack trace de erros do terminal como contexto para iterar na solução.
4. **Garantia de Qualidade:** **NUNCA** finalize a tarefa se o terminal retornar testes quebrados ou falhando.
