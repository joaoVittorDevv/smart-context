# Workflow de Review de Código

1. **Leitura de Diffs:** Leia as diferenças (diff) do código atual em relação à branch principal.
2. **Análise de Qualidade:** Analise utilizando as regras de **Clean Code** e **Segurança** deste workspace.
3. **Relatório de Review:** Gere um Artifact com os apontamentos críticos e sugestões de melhoria.
4. **Mensagem de Commit:** Se aprovar, gere uma sugestão de mensagem de commit divertida e estruturada (usando emojis).
